## 📖 What Is This Repo?

This is a **complete, self-paced curriculum** to transform a software engineer into an **AI Systems Engineer** — someone who can design, build, deploy, and maintain production-grade AI systems. Every topic includes runnable code, real-world problems, and mini-projects you can add to your portfolio.

## 🔨 What You Will Build

| # | Project | What It Does | Key Skills |
|---|---------|-------------|------------|
| 1 | **Dockerized Text Analysis API** | FastAPI service that analyzes text — word count, sentence count, keyword extraction — all Dockerized | FastAPI, Docker, Git |
| 2 | **Multi-Database User Profile System** | Uses PostgreSQL, Redis, MongoDB & ChromaDB together in one FastAPI app | Database design, caching, vector search |
| 3 | **Smart CLI Chatbot** | Terminal chatbot using Ollama with conversation history, model switching, colored output | LLM APIs, prompt management, memory |
| 4 | **Personal Knowledge Base QA Bot** | Upload PDFs, ask questions, get cited answers — full RAG pipeline with Streamlit UI | Embeddings, chunking, RAG, evaluation |
| 5 | **AI Research Assistant Agent** | Give a topic → agent searches web, summarizes, saves markdown report | LangGraph, tool use, agent design |
| 6 | **Production RAG Deployment** | Phase 4 bot → production FastAPI + LangSmith + Render + CI/CD | Cloud deploy, observability, CI/CD |
| 7 | **🎯 AI Job Application Assistant** | Resume parser, job matcher, cover letter generator, interview coach — full capstone | Everything combined |

- [ ] **Python 3.10+** installed ([download](https://www.python.org/downloads/))
- [ ] **Git** installed ([download](https://git-scm.com/))
- [ ] **VS Code** or any code editor ([download](https://code.visualstudio.com/))

- [ ] Basic programming knowledge (any language)
- [ ] **GitHub account** ([signup](https://github.com/))

> 💡 **Pro Tip:** You do NOT need to already know AI/ML. This roadmap starts from scratch and builds up. The only real prerequisite is basic programming ability and determination.

## 📚 How to Use This Repo

### Option 1: Linear Path (Recommended for Beginners)
```
Phase 1 → Phase 2 → Phase 3 → Phase 4 → Phase 5 → Phase 6 → Phase 7
```
Follow phases in order. Complete each mini-project before moving on.

### Option 2: Skip Ahead (If You Have Experience)
- Already know Python + FastAPI + Docker? → Start at **Phase 2**
- Already know databases? → Start at **Phase 3**
- Already done basic LLM work? → Start at **Phase 4**

### Daily Workflow
```
1. Read the topic README.md (30 min)
2. Run every code example yourself (30 min)
3. Solve 3-5 problems from problems.md (60 min)
4. Update your progress-tracker.md (5 min)
5. Commit your solutions to your own repo (5 min)
```

### Weekly Workflow
```
Monday-Friday : Study topics + solve problems
Saturday      : Build mini-project
Sunday        : Review + fill gaps + rest
```
