## 🎯 Why This Phase?

Before you build AI systems, you need solid engineering foundations. You can't deploy an LLM app if you don't understand APIs, containers, or version control. This phase ensures you have the **developer toolkit** every AI Systems Engineer needs.

> ⚠️ **Warning:** Skipping this phase is the #1 reason people struggle later. Even if you know some of these topics, do the problems to validate your knowledge.

## 🧭 Learning Path

```
Week 1: Python for AI (3 days) → FastAPI Basics (3 days)
         └── Practice: Build small API endpoints

Week 2: Git & GitHub (2 days) → Linux & Terminal (2 days) → Docker (starts)
         └── Practice: Create GitHub repo, write shell scripts

Week 3: Docker (continues, 2 days) → Mini Project (4 days)
         └── Build: Dockerized Text Analysis API
```

---

## ✅ Phase Completion Criteria

Before moving to Phase 2, make sure you can:

- [ ] Write Python code with type hints, async/await, and data classes
- [ ] Build a REST API with FastAPI and Pydantic models
- [ ] Use Git for version control (branches, commits, PRs)
- [ ] Navigate the terminal and write basic shell scripts
- [ ] Dockerize a Python application and run it with Docker Compose
- [ ] Complete the mini project and push it to GitHub
