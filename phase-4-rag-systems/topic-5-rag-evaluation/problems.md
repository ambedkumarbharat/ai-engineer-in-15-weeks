## Problem 01 🧩
**Difficulty:** 🟢 Easy
### Create a Test Dataset
**Problem:** Create a RAG evaluation dataset with 20 question-answer pairs and known relevant documents. Store as JSON for reuse.

## Problem 02 🧩
**Difficulty:** 🟢 Easy
### Precision@K and Recall@K
**Problem:** Implement precision@K and recall@K from scratch. Evaluate your RAG pipeline's retrieval quality on 20 test queries.

## Problem 03 🧩
**Difficulty:** 🟡 Medium
### RAGAS Integration
**Problem:** Set up RAGAS evaluation on your RAG pipeline. Measure faithfulness, answer relevancy, and context precision on 30 test cases.

## Problem 04 🧩
**Difficulty:** 🟡 Medium
### LLM-as-Judge
**Problem:** Use an LLM to score RAG outputs on a 1-5 scale for: accuracy, completeness, and relevance. Compare with human ratings.

## Problem 05 🧩
**Difficulty:** 🟡 Medium
### A/B Test Framework
**Problem:** Build a framework to A/B test two RAG configurations (e.g., different chunk sizes). Determine which is better with statistical significance.

## Problem 06 🧩
**Difficulty:** 🟡 Medium
### Hallucination Detector
**Problem:** Build a system that detects when the LLM's answer contains information NOT present in the retrieved context (hallucination).

## Problem 07 🧩
**Difficulty:** 🔴 Hard
### End-to-End Evaluation Dashboard
**Problem:** Build a Streamlit dashboard showing RAG performance: retrieval metrics, generation quality, latency distribution, and cost tracking.

## Problem 08 🧩
**Difficulty:** 🔴 Hard
### Regression Testing Suite
**Problem:** Build a RAG regression test suite that runs automatically on every code change and alerts if any metric drops below a threshold.

## Problem 09 🧩
**Difficulty:** 🔴 Hard
### Human Feedback Loop
**Problem:** Build a feedback system where users rate answers (👍/👎). Store ratings, identify weak spots, and use feedback to improve prompts.

## Problem 10 🧩
**Difficulty:** 🔴 Hard
### Multi-Metric Optimization
**Problem:** Build an optimizer that tunes RAG parameters to maximize a composite score: 0.4×accuracy + 0.3×latency_score + 0.2×cost_score + 0.1×coverage.
